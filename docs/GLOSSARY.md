# Atlas Glossary

> Canonical definitions for all terms used in the Atlas Engineering OS. When in doubt, this document is authoritative.

---

## A

**ADR (Architecture Decision Record)**
A document that captures a significant architectural decision, the context that led to it, the options considered, and the consequences. ADRs are immutable records — never deleted, only superseded.

**Agent**
An autonomous AI unit specialized for a specific task domain within Atlas. Agents have roles, tools, memory access, and communication channels. Distinct from a generic LLM call — agents maintain state, follow protocols, and operate within the Agent Contract.

**Agent Contract**
The formal specification of an agent's role, capabilities, constraints, memory access rights, tool permissions, and communication rules.

**Agent Control Plane**
The infrastructure layer that manages agent lifecycle (spawn, monitor, pause, terminate), enforces security policies, and provides observability across all running agents.

**Audit Engine**
The Atlas engine responsible for evaluating software quality, architecture compliance, security posture, documentation completeness, and generating the Engineering Score.

**ABAC (Attribute-Based Access Control)**
Authorization model where decisions are based on attributes of users, resources, and environment, extending RBAC for fine-grained permissions.

---

## B

**Blueprint**
The primary output of the Atlas Blueprint Engine. A comprehensive, structured specification of a software project including vision, architecture, technology stack, agent configuration, test strategy, and deployment plan. Blueprints are machine-readable and human-readable simultaneously.

**Blueprint-First Methodology**
Atlas's core development methodology requiring that a complete Blueprint must be finalized and validated before any significant implementation begins. No code is written without a Blueprint.

**Blueprint Engine**
The Atlas engine that transforms project requirements (gathered through Discovery) into a complete Blueprint document through multi-agent collaboration, research, and synthesis.

---

## C

**Constitution**
A project-level governance document that defines invariants (things that must always be true), prohibitions (things that must never happen), quality standards, and evolution rules for a specific software project. Generated by the Constitution Engine and enforced by all agents.

**Constitution Engine**
The Atlas engine that generates project constitutions based on project type, requirements, team preferences, and industry standards.

**CQRS (Command Query Responsibility Segregation)**
An architectural pattern separating read (Query) and write (Command) operations, allowing independent optimization of each path. Used throughout Atlas for scalability.

---

## D

**Decision Engine**
The Atlas engine that facilitates architectural decision-making, generates ADR drafts based on decision context, analyzes trade-offs, and learns from past decisions across projects.

**Discovery**
The first phase of the Blueprint-First Methodology, where the Discovery Agent conducts an intelligent, Socratic dialogue to elicit, clarify, and validate all project requirements.

**Discovery Agent**
The specialized agent that conducts the Discovery phase through intelligent questioning, disambiguation, and requirement validation.

---

## E

**Engineering Score**
A composite quality metric (0-100) calculated by the Audit Engine across multiple dimensions: Architecture Quality, Code Quality, Security, Performance, Documentation, Testing, Maintainability, and AI Integration Quality.

**Evolution Engine**
The Atlas engine responsible for continuous improvement recommendations, retrospective analysis, technical debt management, and long-term evolutionary strategy for a project.

**Episodic Memory**
Memory of specific past events, interactions, and outcomes, stored time-ordered in PostgreSQL and queryable for context retrieval.

---

## F

**Flow**
A defined sequence of agent interactions and operations that accomplishes a specific Atlas capability (e.g., Blueprint Generation Flow, Audit Flow).

---

## G

**Graph Engine**
The Atlas engine wrapping Neo4j that manages the Knowledge Graph — entity creation, relationship management, subgraph queries, and GraphRAG retrieval.

**GraphRAG**
Graph-based Retrieval-Augmented Generation: a technique combining vector search with Knowledge Graph traversal to provide richer, more accurate context to LLMs than standard RAG.

---

## K

**Knowledge Graph**
The core memory structure in Atlas that represents projects, technologies, decisions, relationships, and concepts as nodes and edges in a Neo4j graph database, enabling multi-hop reasoning.

---

## L

**LangGraph**
The graph-based agent orchestration framework used by Atlas for stateful multi-agent workflows. Provides checkpointing, cycles, and deterministic state machines.

**Living Documentation**
Documentation that is continuously updated by Atlas agents as code evolves, ensuring docs never become stale.

---

## M

**MCP (Model Context Protocol)**
An open standard protocol for connecting AI applications to external data sources and tools via a client-host-server architecture using JSON-RPC 2.0.

**MCP Engine**
The Atlas engine that discovers, registers, manages, routes requests to, and monitors health of MCP servers available to Atlas agents.

**Memory Engine**
The Atlas engine that manages all three tiers of project memory (Episodic, Semantic, Procedural) and coordinates retrieval, consolidation, and forgetting.

**Monorepo**
A single repository containing all Atlas code — applications, packages, engines, and integrations — managed by Nx for intelligent builds and tooling.

---

## O

**Orchestrator Agent**
The master coordinator agent in Atlas that routes tasks to specialist agents, manages workflow state, handles escalations, and synthesizes results.

---

## P

**Planning Engine**
The Atlas engine responsible for project planning, sprint decomposition, task dependency resolution, effort estimation, and roadmap management.

**Procedural Memory**
Memory of how to accomplish tasks — stored as vector embeddings of successful action sequences, tool-calling patterns, and proven strategies.

**Prompt Engine**
The Atlas engine that generates optimized, versioned system prompts for each agent, conducts A/B testing of prompt variants, and learns from performance feedback.

---

## R

**RBAC (Role-Based Access Control)**
Authorization model where permissions are assigned to roles, and users are assigned roles. Atlas's base authorization model.

**Red Team**
The systematic adversarial testing process where the Red Team Engine attempts to break, exploit, or subvert Atlas components and generated code.

**Red Team Engine**
The Atlas engine that conducts security adversarial testing including prompt injection, jailbreaking attempts, vulnerability scanning, and security regression testing.

**Research Engine**
The Atlas engine that conducts continuous research including state-of-art discovery, academic paper analysis, benchmark evaluation, and technology landscape monitoring.

**RFC (Request for Comments)**
A detailed technical specification document used to define protocols, formats, and systems within Atlas. RFCs are finalized through review, distinct from ADRs which record final decisions.

---

## S

**Score Engine**
The component responsible for calculating, storing, and trending the Engineering Score. Sub-component of the Audit Engine.

**Semantic Memory**
Persistent factual knowledge about a project, stored in the Knowledge Graph (Neo4j) as typed entities and relationships.

**Simulation Engine**
The Atlas engine that models what-if scenarios, validates architectural decisions through simulation, estimates outcomes, and quantifies risks.

**Skill**
A reusable, composable capability that Atlas agents can acquire and execute. Managed by the Skill Engine. Example skills: "Write OpenAPI spec", "Generate Terraform module", "Analyze SQL query performance".

**Skill Engine**
The Atlas engine that manages the skill registry, discovers applicable skills for a task, composes skills into workflows, and learns new skills from agent experiences.

---

## T

**Telemetry**
The collection and transmission of observability data (traces, metrics, logs) via OpenTelemetry to the Atlas observability stack.

---

## V

**Vector Database**
A database optimized for storing and searching high-dimensional vector embeddings. Atlas uses pgvector for tight PostgreSQL integration and Qdrant for dedicated high-performance vector operations.
